--------------------------------------------------------------------------------
1. SCREEN PURPOSE
--------------------------------------------------------------------------------

Display all current stock batches (inventory) for the workspace.

Responsibilities:
  • Load stock batches created from purchases
  • Show key info per batch: Product, Qty, Unit, ReceivedDate, ExpiryDate
  • Filter/sort options (by product, expiry status, received date)
  • Tap a batch to see detailed view (read-only in v1)
  • Show counts: Total products, Total qty, Expiring soon warnings

This screen is the primary inventory status dashboard and entry point for inventory actions (future: quick actions, adjustments).

--------------------------------------------------------------------------------
2. SCREEN ROUTES & FLOWS
================================================================================

Entry Points:
  MainMenu.btnInventory → Navigate(StockBatchList)
  PurchaseEntry.btnCompletePurchase (after purchase) → Navigate(StockBatchList)
  (Future) QuickActions result → Navigate(StockBatchList)

Exit Routes:
  [Tap batch]       StockBatchList.galBatches.OnSelect → Navigate(StockBatchDetail, {drvBatchId: ThisItem})
  [Navigation]      StockBatchList.cmpHeader.btnBack → Navigate(MainMenu)
  [Menu]            StockBatchList.cmpHeader.btnMenu → Open cmpFlyoutMenu

Associated Flows:
  (No flows triggered by this screen; read-only)

Background Data Activities (Asynchronous):
  • Periodic refresh: Every 30s (or on screen focus), refresh colStockBatches to show latest quantities
    Handles: Stock decrements from sales (Vertical 2), event finalization (future)

Screen Display Flow:
  1. User navigates to Inventory
  2. Screen OnVisible loads colStockBatches (active batches with Qty > 0)
  3. Gallery renders by product; shows batch summary
  4. Optional: Filter by expiry status or product
  5. User taps a batch → DetailView with read-only batch info

Error Flow:
  • Zero stock batches → Show empty state with helpful message
  • Dataverse query timeout → Show error with retry button
  • Old app session → Cache may be stale; refresh button available

3. SCREEN STATE & COLLECTIONS
================================================================================

Global Variables (Inherited):
  gblWorkspaceId: Text (Set by Home screen; REQUIRED)

Screen-Local Variables:
  varLoadingBatches: Boolean
    • true while colStockBatches is loading
    • Used to show loading spinner / hide gallery

  varSelectedFilter: Text
    • "All" | "ExpiringIn7Days" | "Expired" | "NoExpiry"
    • Controls which batches are displayed in gallery
    • Default: "All"

  varSelectedProduct: Text (optional filter)
    • GUID of selected ProductVariant (if user filters by product)
    • Blank() for no filter
    • Used to show batches of only one product

  varAutoRefreshEnabled: Boolean
    • true to enable periodic refresh (every 30s)
    • false during manual filter operations (avoid interruption)
    • Default: true

Collections:
  colStockBatches: Table (raw from Dataverse)
    • Filter(StockBatch, Workspace = gblWorkspaceId AND QtyRemaining > 0)
    • Shape: {StockBatchId, ProductVariant.{Name, Unit, LastSellPrice}, QtyReceived, QtyRemaining, ReceivedDateTime, ExpiryDate}
    • Sorted: ExpiryDate ascending (expires first on top)
    • Populated on OnVisible + periodic refresh + filter changes
    • Note: QtyRemaining > 0 hides fully-consumed batches (future: option to show all)

  colStockBatchesFiltered: Table (derived from colStockBatches)
    • Applied filters: Expiry status, product selection
    • Items shown in gallery
    • Recomputed when varSelectedFilter or varSelectedProduct change

  colExpiryStatuses: Table (reference data for filter chips)
    • Hardcoded: {StatusKey: "All", Label: "All"}, {StatusKey: "ExpiringIn7Days", Label: "Expiring in 7 days"}, etc.
    • Used to populate filter chip gallery

  colProductsInBatches: Table (derived)
    • Distinct ProductVariant names from colStockBatches
    • Used for optional product filter dropdown (future; skip in v1)

  colSummaryStats: Table (single row, computed)
    • TotalProducts: CountRows(Distinct(colStockBatches, ProductVariant.ProductVariantId))
    • TotalQty: Sum(colStockBatches, QtyRemaining)
    • ExpiringCount: CountRows(Filter(colStockBatches, ExpiryDate < AddDays(Today(), 7) AND ExpiryDate > Today()))
    • ExpiredCount: CountRows(Filter(colStockBatches, ExpiryDate < Today()))

5. INTERACTION LOGIC (Detailed)
================================================================================

StockBatchList.OnVisible:
  // Initialize on screen load
  Set(varLoadingBatches, true);
  Set(varSelectedFilter, "All");
  Set(varSelectedProduct, Blank());
  Set(varAutoRefreshEnabled, true);
  
  // Load raw stock batches
  ClearCollect(colStockBatches,
    Sort(
      Filter(
        StockBatch,
        Workspace = {Value: gblWorkspaceId},
        QtyRemaining > 0  // Hide fully-consumed batches
      ),
      ExpiryDate,  // Expires first on top
      Ascending
    )
  );
  
  // Load filter reference data
  ClearCollect(colExpiryStatuses,
    Table(
      {StatusKey: "All", Label: "All"},
      {StatusKey: "ExpiringIn7Days", Label: "Expiring in 7 days"},
      {StatusKey: "Expired", Label: "Expired"},
      {StatusKey: "NoExpiry", Label: "No expiry"}
    )
  );
  
  // Apply initial filters (none) and compute filtered collection
  Set(varLoadingBatches, false);
  UpdateFilteredBatches()

UpdateFilteredBatches():
  // Helper function; apply varSelectedFilter and varSelectedProduct to colStockBatches
  
  ClearCollect(colStockBatchesFiltered,
    Filter(
      colStockBatches,
      (
        If(varSelectedFilter = "All",
          true,
          If(varSelectedFilter = "ExpiringIn7Days",
            ExpiryDate >= Today() AND ExpiryDate <= AddDays(Today(), 7),
            If(varSelectedFilter = "Expired",
              ExpiryDate < Today(),
              If(varSelectedFilter = "NoExpiry",
                IsBlank(ExpiryDate),
                true  // fallback
              )
            )
          )
        )
      )
      AND
      (
        If(IsBlank(varSelectedProduct),
          true,
          ProductVariant.ProductVariantId = varSelectedProduct
        )
      )
    )
  );
  
  // Compute summary stats
  ClearCollect(colSummaryStats,
    Table({
      TotalProducts: CountRows(Distinct(colStockBatches, ProductVariant.ProductVariantId)),
      TotalQty: Sum(colStockBatches, QtyRemaining),
      ExpiringCount: CountRows(Filter(colStockBatches, ExpiryDate < AddDays(Today(), 7) AND ExpiryDate > Today())),
      ExpiredCount: CountRows(Filter(colStockBatches, ExpiryDate < Today()))
    })
  )

galExpiryFilter.OnSelect (filter chip):
  Set(varSelectedFilter, ThisItem.StatusKey);
  UpdateFilteredBatches()

galStockBatches.OnSelect (batch card):
  Navigate(StockBatchDetail, {drvBatchId: ThisItem.StockBatchId})

btnRefresh.OnSelect:
  Set(varLoadingBatches, true);
  Refresh(colStockBatches);
  Refresh(colStockBatchesFiltered);
  Set(varLoadingBatches, false);
  Notify("Inventory refreshed", NotificationType.Success)

btnRegisterPurchase.OnSelect:
  Navigate(PurchaseEntry)

6. DATA DEPENDENCIES (Vertical 1 Phase B - Must Exist)

Required Tables:

  • ProductVariant (referenced by StockBatch)

    • UnitId (GUID PK)
    • Name (PrimaryName)

Required Relationships:
  • StockBatch.ProductVariant → ProductVariant (N:1)
  • ProductVariant.Unit → Unit (N:1)

Recommended Indexes (Performance):
  • StockBatch: (Workspace, QtyRemaining) — for filtering active batches
  • StockBatch: (Workspace, ProductVariant, ExpiryDate) — for FIFO queries (future sales)
  • StockBatch: (ExpiryDate) — for "expiring soon" filtering

7. ERROR HANDLING & EDGE CASES
================================================================================

Scenario: StockBatch query times out
  Detected In: colStockBatches load fails; varLoadingBatches remains true
  Response: Show loading spinner indefinitely; user clicks "Refresh" button
  Recovery: Manual refresh button triggers Refresh(colStockBatches)
  Recommendation: Implement timeout + error message after 10s (ISS-021)

Scenario: No stock batches exist (newly onboarded workspace)
  Detected In: colStockBatches.IsEmpty() = true
  Response: Show empty state with "Register a Purchase" button
  Recovery: User navigates to PurchaseEntry; creates purchase; navigates back
  Verification: Gallery populates with new batch

Scenario: All batches filtered out (user selects filter with zero matches)
  Detected In: colStockBatchesFiltered.IsEmpty() = true ; varSelectedFilter <> "All"
  Response: Show empty state with "Clear Filter" button
  Recovery: User clicks filter chip to return to "All"

Scenario: Batch QtyRemaining becomes 0 after a sale (future Vertical 2)
  Current Behavior (v1): Batch remains visible (QtyRemaining > 0 filter excludes fully-consumed)
  Future Behavior (Vertical 2): Decrement on sale → batch hidden automatically
  Design: Gallery auto-refresh on sale completion (async notification from sales screen)

Scenario: ExpiryDate is in the past (batch expired)
  Detected In: ExpiryDate < Today()
  Response: Red "EXPIRED: mm/dd/yyyy" text + red left border on card
  Recovery: Future action to create InventoryEvent (Waste module, v2)
  User Action: Tap card → StockBatchDetail → "Mark as Waste" (future button)

Scenario: ExpiryDate is within 7 days
  Detected In: ExpiryDate < AddDays(Today(), 7)
  Response: Orange/amber "Expires: mm/dd/yyyy" text + orange left border
  Recommendation: Show icon (⚠) to draw attention

Scenario: ExpiryDate is null (no lifespan configured)
  Detected In: IsBlank(ExpiryDate)
  Response: "No expiry" text in gray + gray left border
  Design: No visual urgency; treat as indefinite shelf-life

Scenario: User has multiple workspaces; views Workspace A inventory, then switches to B
  Expected: Gallery immediately shows Workspace B batches (not Workspace A)
  Actual: On Home (workspace change) → gblWorkspaceId updated → Navigate(StockBatchList) → OnVisible loads correct workspace
  Verification: Filter(colStockBatches, Workspace = {Value: gblWorkspaceId}) filters correctly

Scenario: Batch creation takes 10+ seconds (flow latency)
  Expected: User completes purchase → navigates to Inventory → new batch not yet visible → refreshes → appears
  Design: Toast message on purchase completion: "Stock batch will appear in a moment."
  Recovery: "Refresh" button available; user can manually refresh
  Recommendation: Show loading state or "Syncing..." indicator

Scenario: QtyRemaining is a decimal (e.g., 0.75 kg)
  Display: "0.75 in stock" (formatted number)
  Acceptance: Supported; no truncation

Scenario: Very large numbers (10,000+ units)
  Display: "10,000+" (formatted with thousands separator)
  Acceptance: No special handling; Power Apps handles large numbers

Scenario: Product name is very long (e.g., "Premium Black Beans – Extra Large – Organic – Fair Trade")
  Display: Text wraps to multiple lines in gallery cell
  Recommendation: Truncate on design or set fixed height; avoid overflow
  Future: Add tooltip with full name on hover

8. PERFORMANCE & SCALABILITY
================================================================================

Optimization Notes:
  • Filter by Workspace + QtyRemaining > 0 early (reduces dataset size)
  • Sort by ExpiryDate (prioritizes expiring batches on top)
  • Avoid nested galleries (single galStockBatches for all batches)
  • colStockBatchesFiltered is derived; O(n) filter operation on each change

Load Time Estimate:
  • Screen OnVisible: ~500ms (colStockBatches load from Dataverse)
  • Gallery render (< 50 batches): ~100ms (light; simple cards)
  • Filter change (galExpiryFilter chip tap): ~200ms (UpdateFilteredBatches logic)
  • Refresh button: ~1–2s (Refresh = data re-fetch from server)

Scalability (future):
  • Expected: 100–1000 batches per workspace (multi-store chains)
  • If > 500 batches: Consider pagination
  • If > 1000 batches: Add search/advanced filter (future v2 feature)
  • Current design: No paging; loads all active batches at once

Mobile Considerations:
  • Vertical gallery: scrollable (no horizontal scroll)
  • Card layout: responsive; text wraps
  • Filter chips: scrollable horizontally if many filters
  • Summary row: stacks on small screens (3 cols → 2 cols or 1 col)

Accessibility:
  • Gallery items focusable (keyboard navigation)
  • Card OnSelect understandable without color alone (text + border)
  • All text has sufficient contrast
  • Loading state announced ("Loading inventory...")

Caching Strategy (Optional v1.1):
  • Store colStockBatches in localStorage to speed up repeated navigation
  • Refresh on OnVisible after 1 minute (stale data risk)
  • Not recommended for initial v1 (keep simple; always fresh)

Expiry Status Color Scheme:
  • Expired: Red (#D9534F)
  • Expiring in 7 days: Orange (#F0AD4E)

  • No expiry: Gray (#777)
